10-闭包与作用域链

发表于 2021-09-09 更新于 2021-09-11 分类于 JS 阅读次数：

自由变量：

在作用域内能访问到的变量，但是变量声明在外部

执行上下文：

  • 全局执行上下文，默认的，在浏览器中是 window 对象，并且 this 在非严格模式下指向它。
  • 函数执行上下文，JS 的函数每当被调用时会创建一个上下文。
  • Eval 执行上下文，eval 函数会产生自己的上下文，通常用于静态测试，将外部变量全部打包给内部函数。

作用域链：

变量或者函数或者对象等在创建的时候会依下向上形成一条作用域链

💊This和作用域如何确定

JavaScript 解释阶段便会确定作用域规则，因此作用域在函数定义时就已经确定了，而不是在函数调用时确定，但是执行上下文是函数执行之前创建的。执行上下文最明显的就是 this 的指向是执行时确定的。而**作用域访问的变量是编写代码的结构确定**的。

一.静态作用域链

函数和块的作用域内的变量声明会在作用域 （scope） 内创建一个绑定（变量名绑定到具体的值，也就是 binding），然后其余地方可以引用 （refer） 这个 binding，这样就是静态作用域链的变量访问顺序。

二.闭包

闭包是返回函数的时候扫描函数内的标识符引用，把用到的本作用域的变量打成 Closure 包，放到 [[Scopes]] 里。

const fun = fun1()

三.总结

闭包本质是在一个函数内的函数访问了外函数的作用域，如果没有返回，JS会建立一条静态作用域链将变量串联起来，如果返回了，那么这个函数就会保存外函数的作用域放在[[scopes]]里，全局作用域也会自动保存在里面，
