18-Vue路由守卫

发表于 2021-09-17 更新于 2021-10-02 分类于 Vue

🌈路由守卫调用顺序

  1. 导航被触发。
  2. 在失活的组件里调用 beforeRouteLeave 守卫。
  3. 调用 全局 的 beforeEach 守卫。
  4. 在重用的组件里调用 beforeRouteUpdate 守卫(2.2+)。
  5. 在路由配置里调用 beforeEnter。
  6. 解析异步路由组件。
  7. 在被激活的组件里调用 beforeRouteEnter。
  8. 调用全局的 beforeResolve 守卫(2.5+)。
  9. 导航被确认。
  10. 调用全局的 afterEach 钩子。
  11. 触发 DOM 更新。
  12. 调用 beforeRouteEnter 守卫中传给 next 的回调函数，创建好的组件实例会作为回调函数的参数传入。

请求头(浏览器自动添加)

Origin: xxx.com

Access-Control-Request-Method: Put

Access-Control-Allow-Origin: xxx.com

Access-Control-Allow-Methods: Get,Post,Put

Access-Control-Allow-Credentials: true // 客户端ajax请求时const xhr = new XmlHttpRequest xhr.withCredentials = true

Access-Control-Max-Age: 1998 // 单位s

  • Accept
  • Accept-Language
  • Content-Language

13-promise详解

发表于 2021-09-15 更新于 2021-10-11 分类于 JS

📌补充：

①.Promise 基本原理

🐵Promise 是一个类，在执行这个类的时候会传入一个执行器，这个执行器会立即执行

🙈Promise 会有三种状态

  • Pending 等待
  • Fulfilled 完成
  • Rejected 失败

🙉状态只能由 Pending –> Fulfilled 或者 Pending –> Rejected，且一但发生改变便不可二次修改；

🙊Promise 中使用 resolve 和 reject 两个函数来更改状态；

🐒then 方法内部做但事情就是状态判断

  • 如果状态是成功，调用成功回调函数
  • 如果状态是失败，调用失败回调函数
